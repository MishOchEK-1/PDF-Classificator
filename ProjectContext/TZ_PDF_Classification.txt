
Техническое задание
Проект: AI PDF Document Classification System

1. Цель проекта
Разработать веб-приложение для загрузки PDF-документов и их автоматической классификации с использованием локальной LLM-модели Gemma 4 через Ollama.

Система должна:
- принимать PDF-файлы;
- извлекать текст из PDF;
- классифицировать документ;
- генерировать метки (tags);
- отображать результат пользователю.

----------------------------------------
2. Архитектура системы

Frontend:
- простой веб-интерфейс на HTML/CSS/JS для загрузки PDF;
- отображение результата классификации;
- отображение тегов и confidence score.

Backend:
- Django Backend;
- обработка запросов;
- работа с файлами;
- интеграция с Ollama API;
- генерация prompt для модели.

LLM Layer:
- локально запущенная Gemma 4;
- управление моделью через Ollama.

----------------------------------------
3. Основной pipeline

1. Пользователь загружает PDF.
2. Веб-страница отправляет файл в Django Backend.
3. Backend сохраняет файл во временное хранилище.
4. PDF Parser извлекает текст.
5. Prompt Builder формирует prompt.
6. Backend отправляет запрос в Ollama API.
7. Ollama вызывает Gemma 4.
8. Модель возвращает:
   - document class;
   - tags;
   - confidence score.
9. Backend возвращает JSON.
10. Веб-интерфейс отображает результат.

----------------------------------------
4. Формат ответа модели

Пример JSON:

{
  "class": "technical_documentation",
  "tags": [
    "python",
    "backend",
    "api"
  ],
  "confidence": 0.92
}

----------------------------------------
7. Основные требования

Функциональные:
- загрузка PDF;
- обработка PDF;
- классификация документа;
- генерация tags;
- отображение результата.

Нефункциональные:
- локальный запуск модели;
- отсутствие зависимости от внешнего API;
- модульная архитектура;
- расширяемость списка классов.

----------------------------------------
8. MVP Scope

Для первой версии:
- только PDF;
- prompt-based classification;
- без fine-tuning;
- локальная inference-модель;
- REST API communication.
